Multi-Game Registration — Layout Fit

Players can register for multiple games at once. Here's how each layout handles that.

A — Wizard ⚠️ Gets awkward

Step 1 becomes a game picker, but selected games need to carry through all steps — the progress bar gets confusing when the "per-game" and "per-player" steps apply to multiple games at once.

Problem: Step confusion
Awkward edge cases
  • Does "position" apply to all games or per-game?
  • Payment screenshot: one per game or one total?
  • Adding a game mid-flow breaks the wizard state
  • Step 3 header: "Players for which game?"
  • Progress bar misleads — "2 of 4 steps" but really 2×4
Wizard works best for a single linear item. Multi-game breaks the mental model.

B — Split Panel ✅ Natural fit

Left panel becomes a "cart" — shows all selected games, running total, remove buttons. Right panel stays focused on the form. Adding games just updates the left side.

Tradeoffs
  • Page gets very long with 2+ games + group/team mode
  • Running total only visible at the very bottom
  • No persistent summary — easy to lose track
