# Quick Start

## 1. Supabase Setup (5 min)

```bash
# Go to supabase.com
# Create 2 projects: volleyball-staging, volleyball-prod
# Run all 5 migrations in SQL Editor (in order)
```

## 2. Environment Variables

```bash
# .env.local
NEXT_PUBLIC_SUPABASE_URL=https://your-staging-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-staging-anon-key
NEXT_PUBLIC_SITE_URL=http://localhost:3000
```

## 3. Run Dev Server

```bash
npm run dev
# Open http://localhost:3000
# You'll be redirected to /auth automatically
```

## 4. Test Login

| Email | Password | Expected |
|-------|----------|----------|
| anything@test.com | password123 | → `/player` |
| admin@whitelist.com | password123 | → `/admin` (if added to role_whitelist) |

## Database Quick Reference

### Create a Schedule (admin only)

```sql
INSERT INTO public.schedules (
  title, start_time, end_time, venue, max_players, required_level, created_by
) VALUES (
  'Friday Night Volleyball',
  '2024-03-22 19:00:00+0',
  '2024-03-22 21:00:00+0',
  'City Sports Center',
  12,
  'intermediate',
  'your-user-id'
);
```

### Register a Player (player or admin)

```sql
INSERT INTO public.registrations (
  schedule_id, registered_by, player_id, team_preference
) VALUES (
  'schedule-uuid',
  'registering-user-id',
  'player-id',
  'shuffle'
);
-- qr_token auto-generated by trigger
```

### Whitelist an Admin

```sql
INSERT INTO public.role_whitelist (email, role) VALUES
  ('your@admin.email', 'admin');
-- They'll get admin role on next signup
```
